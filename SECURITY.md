# Security Policy

## Supported Versions

Apenas a versão estável mais recente do projeto recebe atualizações de segurança. Como o projeto está em desenvolvimento ativo (pré-1.0 conceitual, versionado como V1.x), versões anteriores não são mantidas retroativamente.

| Version | Supported          |
| ------- | ------------------ |
| 1.20.x  | :white_check_mark: |
| 1.19.x  | :x:                |
| < 1.19  | :x:                |

## Reporting a Vulnerability

Se você encontrar uma vulnerabilidade de segurança neste projeto, **não abra uma issue pública**. Em vez disso, reporte de forma privada através do **GitHub Security Advisories**:

1. Vá até a aba **Security** do repositório
2. Clique em **Report a vulnerability**
3. Descreva o problema com o máximo de detalhes possível (passos para reproduzir, impacto potencial, versão afetada)

### O que esperar
- **Confirmação de recebimento:** em até 5 dias úteis
- **Atualização de status:** a cada 7–10 dias até a resolução
- **Se aceita:** a correção será desenvolvida em branch privada, testada e lançada como patch; o reporte será divulgado publicamente após o release, com crédito ao autor (caso deseje)
- **Se recusada:** você receberá uma explicação do motivo (ex: não é uma vulnerabilidade real, está fora do escopo do projeto, ou é um comportamento esperado)

Como este é um projeto pessoal em desenvolvimento, não há SLA formal, mas relatórios de segurança são tratados com prioridade acima de outras issues.
